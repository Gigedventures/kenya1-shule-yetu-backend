# Codex Build Protocol — Shule Yetu

This document defines NON-NEGOTIABLE rules for all code generated by Codex
inside the Shule Yetu / Kenya 1 platform.

Codex must follow these rules strictly.

---

## 1. Architecture Rules (MANDATORY)

- Project is a Laravel 10 Modular Monolith
- Shule Yetu modules live under: app/Modules/ShuleYetu
- All Shule Yetu database tables MUST be prefixed with: shule_
- Multi-school tenancy is enforced at all times
- Every tenant record MUST belong to exactly one school_id
- No cross-school data access is allowed

---

## 2. Tenancy Rules (FAIL-CLOSED)

- SchoolContext MUST be used for all tenant-scoped operations
- If SchoolContext is missing:
  - Reads MUST return no records
  - Writes MUST throw an exception
- All tenant models MUST extend BaseShuleModel
- No model may bypass ScopedToSchool

---

## 3. ID & Key Rules

- Tenant tables use UUID primary keys
- users.id remains BIGINT
- Foreign keys MUST match referenced column type
- school_id is UUID everywhere in tenant tables

---

## 4. Model Rules

Every tenant model MUST:
- extend BaseShuleModel
- include school_id
- rely on ScopedToSchool for isolation
- throw on cross-school relationship mismatch

---

## 5. Migration Rules

- All module migrations live in:
  app/Modules/ShuleYetu/Database/Migrations
- Root database/migrations contains ONLY Laravel core migrations
- All migrations must include:
  - indexes
  - foreign keys
  - unique constraints where applicable

---

## 6. RBAC Rules

- RBAC is school-scoped
- Permissions are checked using:
  $user->hasPermission('permission.name')
- No global role leakage
- Superadmin behavior must be explicit and controlled

---

## 7. Filament Rules

- All Filament resources must respect tenancy automatically
- No unscoped queries allowed
- Authorization must use RBAC permissions
- No hard-coded role checks

---

## 8. Testing Rules

Every module MUST include tests for:
- Cross-school isolation
- Business rule enforcement
- Duplicate prevention
- Fail-closed behavior

No module is considered complete without passing tests.

---

## 9. Forbidden Actions

Codex MUST NOT:
- Reintroduce legacy sy_ tables
- Add global access shortcuts
- Disable tenancy
- Create demo-only logic
- Skip tests
